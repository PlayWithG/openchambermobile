# Task Checklist: OpenChamber Android App

## Overview

This task checklist implements the android-app change following the MVP priority order defined in REQ-07. Tasks are ordered by dependency — earlier tasks must be completable before later ones start.

**Priority Order:**
- **P1**: Connect to remote server (REQ-01 Capacitor wrapper + REQ-03 session persistence)
- **P2**: Usable foreground session (REQ-02 connectivity baseline + REQ-05 layout)
- **P3**: Recover from background/network (REQ-02 full lifecycle + REQ-04 status UI)
- **P4**: Comfortable on S24 Ultra (REQ-05 full validation)

---

## P3: Recover from Background/Network

### T-11: Implement Background/Foreground Handling

**Description:** Complete the connectivity manager's background/foreground lifecycle handling.

**Files to Modify:**
- `packages/ui/src/lib/opencode/connectivity.ts` — Full implementation

**Dependencies:** T-07, T-08

**Acceptance Criteria:**
- [ ] On `appStateChange` to background: transition to `paused`, abort SSE
- [ ] On `appStateChange` to foreground: validate liveness before reconnect
- [ ] If liveness check passes: restore existing session
- [ ] If liveness check fails: discard stale session, create new connection
- [ ] Connection restored within 5 seconds under normal conditions

**Priority:** P3

---

### T-12: Implement Network Change Handling

**Description:** Handle network type transitions (Wi-Fi ↔ mobile data) with immediate reconnection.

**Files to Modify:**
- `packages/ui/src/lib/opencode/connectivity.ts` — Network change handling

**Dependencies:** T-07, T-11

**Acceptance Criteria:**
- [ ] On network change: abort current SSE immediately (no heartbeat wait)
- [ ] Reconnect within 5 seconds on new network
- [ ] No duplicate messages received (event pipeline handles no-op)
- [ ] State transitions: `reconnecting` → `connected`

**Priority:** P3

---

### T-13: Implement Exponential Backoff

**Description:** Add exponential backoff with jitter for server unreachable scenarios.

**Files to Modify:**
- `packages/ui/src/lib/opencode/connectivity.ts` — Backoff implementation

**Dependencies:** T-07

**Acceptance Criteria:**
- [ ] Initial delay: 1000ms
- [ ] Multiplier: 2x per attempt
- [ ] Cap: 30000ms
- [ ] Jitter applied to prevent thundering herd
- [ ] Retry throttled during background (respects cap interval)
- [ ] Backoff resets on successful connection

**Priority:** P3
